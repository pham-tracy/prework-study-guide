HTML

  • Gives content structure
  • The head element contains information about the webpage. This includes metadata that helps browser render the page correctly.
  • < meta > includes elements like character set, description, keywords, author, viewport
  • The body element represents the visible content shown to the user
  • The syntax is < element attribute = "value" >< /element >
  • Begins with < !DOCTYPE html >
  • < base > defines base URL for webpage
  • script element embeds client-side scripts like JavaScript
  • Child elements: elements nested inside others, which are known as Parent elements
  • < div > : creates division
  • img src = source/location of file; alt = text string describing image/functionality

CSS

  • Gives content style
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element.
  • Class selectors allow you to style specific elements with an assigned class
  • Class selectors begin with .
  • margin: top/bottom left/right
  • Linking external CSS stylesheet in HTML code: < link rel="stylesheet" href= file location>
  • External CSS stylesheets should be linked in head section
  • selector {property:value}
  • CSS box model: content -> padding (space around content inside of element) -> border -> margin (space around outside of element)
  • border syntax: width style color
  • box shadow: shadow effets around frame. X & Y offsets relative to element, blur, spread radius, color. X-axis (right of box), Y-axis (below box)

Git

  • git status: checks what branch you are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout main: returns to main branch
  • Git flow: go to feature branch.
  • git add -A: stage changes
  • git commit -m "< your message here >": add commit message to describe what changes were made
  • git pull origin main: makes sure local current branch is up to date before pushing changes. This pulls in base branch main to the feature branch and syncs the two.
  • git push origin < branch name > : push up changes to remote repository in GitHub
  • Go to GitHub repo then to working branch. Click "Pull Requests" tab -> "New pull request"
  • Compare two branches you want to merge. Base: main (what we want to merge into), Compare: feature branch you want to merge into main.
  • Title and brief description for pull request > "Create pull request" > PR is created
  • Click "Merge pull request" to merge. Confirm merge by navigating to main branch in GitHub
  • Update local repository with new merged-in feature: git checkout main > git pull origin main
  • Close issues as needed in GitHub

JavaScript

  • Handles logic and adds interactivity to website.
  • Links by adding opening and closing tag script src ="location". Link in body of HTML code
  • Variables: containers that store values
  • Use var keyword to declare variable
  • Variables are case sensitive and are retrieved by calling variable name
  • Comments: /* Comment multiple lines */ or // Comment single line
  • Strings: text data, must have single quotation marks
  • Numbers: numbers, no quote marks needed
  • Boolean: true/false
  • Array: structure that allows storage of multiple values in single reference
  • Object: anything. Everything in JavaScript is an object and can be stored as variable
  • Strict equality (===): performs test to see if two values are equal. Returns Boolean true/false result
  • Not, Does-not-equal (!, !==): returns logically opposite value of what it precedes. Turns true into false, etc. When used with Equality negation operator, it tests whether two values are not equal.
  • Control flow: order which a computer executes a code. In JavaScript, it is read top to bottom. Can be interrupted with conditional statements and loops to control code execution order
  • Conditional statements: If statement: code executes if statement meets a condition or is tru. Computer will check for truthy
  • Falsy: 0, -0, On, "", null, undefined, NaN, Boolean data type with value false
  • Truthy:All other values, including Boolean with value true
  • Can expand and use if, else if: if (first condition) { //code if first condition truthy } else if (second condition) { //code if second condition truthy }
  • Array syntax: var name = ["x", "y", "z", "etc"]; and can hold various types of data like strings, numbers, etc.
  • Array index begins with 0 and are sequential. First item index =0, second =1, etc.
  • For loops: uses predictable pattern of indices to perform task on all array items by allowing single code block to execute over and over. To interrupt, if conditional statement needs to be used.
  • For loops: requires array's index and array's length. Length is built in property, ex: variablename.length
  • x++ is shorthand for x+1
  • Chrome DevTools: Right click, Inspect to view code. Command+Option+I on Mac
